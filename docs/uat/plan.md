# ユーザー受入テスト（UAT）計画書

## 1. 概要

### 1.1 目的

本ドキュメントは、福祉相談業務AI支援システム「Kyugo AI」の納品前ユーザー受入テスト（UAT）の計画を定めるものです。実際の福祉相談員がシステムを操作し、業務要件を満たしているかを検証します。

### 1.2 対象システム

- **システム名**: Kyugo AI（福祉相談業務AI支援システム）
- **URL**: https://kyugo-ai-2knyenyska-an.a.run.app
- **主要機能**: ケース管理、相談記録（テキスト/音声）、AI分析、支援計画書作成、モニタリングシート、法令・制度検索、CSVエクスポート

### 1.3 UATの範囲

| 対象 | 範囲 |
|------|------|
| テスト対象 | 全画面・全機能（8画面、14操作フロー） |
| テスト実施者 | 福祉相談員（2-3名推奨）、管理者（1名） |
| テスト環境 | 本番環境（テスト用アカウントを発行） |
| 対象外 | 性能テスト、セキュリティ診断（別途実施済み） |

---

## 2. テスト環境の準備

### 2.1 アカウント発行

UAT実施者ごとにGoogleアカウントのメールアドレスを許可リストに追加します。

**手順（管理者が実施）**:

1. 管理者アカウントでログイン
2. サイドバー「アクセス設定」を開く
3. 「ログイン許可設定」タブで実施者のメールアドレスを追加
4. 実施者にログインURLを共有

### 2.2 デモデータの投入

テスト開始前に、AI分析済みのサンプルデータを投入します。

```bash
# デモデータ投入コマンド（管理者が実施）
npm run seed:demo -- <staffId>
```

**投入されるデータ**:

| ケース | 相談者 | 相談記録 | AI分析 | 支援計画書 |
|--------|--------|---------|--------|-----------|
| D-2026-001 | 佐藤 美咲（67歳） | 1件 | 済 | 下書きあり |
| D-2026-002 | 田中 健一（51歳） | 2件 | 済 | なし |

### 2.3 事前確認チェックリスト

- [ ] ヘルスチェック正常（`/health` にアクセスして確認）
- [ ] 全テスト実施者がログインできることを確認
- [ ] デモデータがダッシュボードに表示されることを確認
- [ ] マイクの利用許可がブラウザに設定済み（音声テスト用）
- [ ] Excel/スプレッドシートが利用可能（CSVエクスポート確認用）
- [ ] ネットワーク接続が安定していること

### S2: 新規ケース作成〜テキスト相談記録〜AI分析

**目的**: ケース作成から相談記録入力、AI分析までの一連のフローを確認する

| # | 操作 | 期待結果 | 合否 |
|---|------|---------|------|
| 1 | 「＋ 新規ケース」をクリック | ケース作成フォームが表示される | □ |
| 2 | 相談者氏名・ID・生年月日を入力し作成 | ケース詳細画面に遷移する | □ |
| 3 | 「＋ 新規相談記録」をクリック | 相談記録入力フォームが表示される | □ |
| 4 | 相談内容をテキスト入力し記録 | 記録が保存され一覧に表示される | □ |
| 5 | AI分析の完了を待つ（5-15秒） | ローディング表示後、分析結果が表示される | □ |
| 6 | AI分析結果を確認 | 要約（200文字以内）が表示される | □ |
| 7 | 支援メニュー提案を確認 | 関連度スコア付きで支援メニューが提案される | □ |

**テスト用相談内容（コピペ用）**:
```
60代の男性。3ヶ月前に勤務先の工場が閉鎖となり失業。雇用保険の受給期間が終了し、
現在は貯蓄を切り崩して生活している。家賃の支払いが2ヶ月滞納となっており、
大家から退去を求められている。高血圧の持病があるが、国民健康保険料も未納で
医療機関の受診を控えている。独居で身寄りはなく、近隣との交流もほとんどない。
```

## 4. 評価基準

### 4.1 合格判定

| 判定 | 基準 |
|------|------|
| **合格** | 全シナリオ（S1-S8）の全項目が「合」 |
| **条件付き合格** | 軽微な不具合が3件以下かつ業務遂行に支障なし |
| **不合格** | 業務遂行に支障のある不具合が1件以上 |

### 4.2 不具合の分類

| 重要度 | 定義 | 例 |
|--------|------|-----|
| **致命的** | 業務が完全に停止する | ログインできない、データが保存されない |
| **重大** | 主要機能が使用不能 | AI分析が動作しない、音声録音ができない |
| **軽微** | 操作は可能だが改善が望ましい | 表示の崩れ、文言の不適切さ |
| **要望** | 機能としては問題ないが追加希望 | 操作性の改善提案 |

## 6. スケジュール提案

### 6.1 全体スケジュール（3日間）

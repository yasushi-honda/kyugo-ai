# ユーザー管理

## 概要

kyugo-ai の職員アカウントは Firebase Authentication + Firestore の `staff` コレクションで管理される。初回ログイン時に自動プロビジョニングされ、管理者がロール・有効/無効を管理する。

## アカウントのライフサイクル

```
メール許可リストに追加 → 初回Googleログイン → 自動プロビジョニング（staff作成）→ 利用開始
                                                                              ↓
                                               退職時: 管理者がアカウント無効化
```

## 新規職員の追加

### 1. メール許可リストに追加

システム管理画面（アクセス設定）で以下のいずれかを設定：

- **個別メールアドレス**: `yamada.taro@example.lg.jp`
- **ドメイン許可**: `example.lg.jp`（そのドメインの全メールを許可）

### 2. 初回ログイン

職員がシステムURLにアクセスし、Googleアカウントでログインすると、自動的に：
- Firebase Authentication にユーザーが作成される
- Firestore `staff` コレクションにドキュメントが作成される（role: `staff`）

### 3. 管理者権限の付与

管理者がシステム管理画面（アクセス設定 → 職員管理）で対象職員のロールを `admin` に変更する。

## 退職者のアカウント無効化

### 手順

1. 管理画面 → アクセス設定 → 職員管理
2. 対象職員の「無効化」をONにする
3. （推奨）許可メールリストからも削除する

### 注意事項

- **最後の管理者は無効化できない**（システムで自動防止）
- 無効化された職員はログインしてもアクセスが拒否される
- 担当ケースのデータは保持される（管理者が別の職員に再割り当て可能）

## ロール一覧

| ロール | 権限 |
|--------|------|
| `staff` | 自分の担当ケースの閲覧・編集、相談記録の作成 |
| `admin` | 全ケースの閲覧、職員管理、アクセス設定、AIリトライ |

## API（管理者用）

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/admin-settings/staff` | 職員一覧 |
| PATCH | `/api/admin-settings/staff/:id` | ロール変更・無効化 |
| GET | `/api/admin-settings/allowed-emails` | 許可メール設定取得 |
| PUT | `/api/admin-settings/allowed-emails` | 許可メール設定更新 |

## トラブルシューティング

### 職員がログインできない

1. 許可メールリストにメールアドレスまたはドメインが含まれているか確認
2. Firebase Authentication でユーザーが無効になっていないか確認
3. `staff` コレクションで `disabled: true` になっていないか確認

### 管理者権限が付与できない

- 自分自身の権限変更はできない（別の管理者に依頼）
- 対象職員がシステムに一度もログインしていない場合は、まずログインを促す

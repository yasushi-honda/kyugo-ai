# ADR-004: 音声文字起こし方式の選定

## ステータス: 承認済み

## コンテキスト

福祉相談業務では訪問・窓口での相談内容を音声で記録し、文字起こし・要約・支援メニュー提案を行う必要がある。
GCPには以下の選択肢がある：
1. Cloud Speech-to-Text V2 + Gemini（2段階処理）
2. Gemini 2.5 Flash マルチモーダル音声入力（1段階処理）

## 決定

**Gemini 2.5 Flash のマルチモーダル音声入力を採用する。**

### 理由

| 観点 | Speech-to-Text + Gemini | Gemini 音声入力のみ |
|------|------------------------|-------------------|
| コスト | ~$0.024/分 + テキスト分析 | **~$0.002/分（約12倍安い）** |
| APIコール | 2回（文字起こし→分析） | **1回** |
| レイテンシ | 2段階の合計 | **1段階のみ** |
| 追加API | Speech-to-Text API有効化が必要 | **不要（既存Vertex AI）** |
| セキュリティ | 同等 | **同等（同一基盤）** |
| 日本語対応 | Chirpモデルで高精度 | Geminiのマルチリンガル対応 |

### トレードオフ
- Speech-to-Textは話者分離（Diarization）がより成熟
- Geminiの音声は最大約8.4時間/100万トークンの制限
- 福祉相談は通常30分〜1時間のため、制限内に十分収まる

## 結果

- Vertex AI（Gemini 2.5 Flash）に音声バイナリを直接送信
- 1回のAPI callで文字起こし＋要約＋支援メニュー提案を一括取得
- 追加APIの有効化不要、既存のWorkload Identity設定をそのまま利用
- コスト大幅削減（約12倍安い）
